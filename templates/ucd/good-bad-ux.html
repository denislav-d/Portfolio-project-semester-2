Good UX design example

Good User Experience – Parking lots lights above parking spaces

Learnability: It is quite easy for users to learn, since everyone knows what the colors green and red correspond with.

Efficiency: The design is super-efficient, because users can see if there is a free parking spot from far away, which saves a lot of time.

Memorability: It is rather hard to forget the design even for a long period of not using it, but even if they do, the learnability speaks for itself.

Errors: Errors in this design are quite rare, but if there is one, it is very easy to recover by just searching for another free parking spot.

Satisfaction: It is very pleasant to use the design, since nobody wants to waste time on such a small thing as searching for a free parking spot.

Bad UX design example

Bad User Experience – Certain type of buttons for traffic lights which pedestrians and cyclists use.

Learnability: It is easy for users to learn, but that is usually after trial and error or when they see how other people use it.

Efficiency: The design is efficient once you have learned how to use it, but you might get frustrated the first time.

Memorability: It is easy to remember what to do, since it is just touching it once with your hand.

Errors: Errors in this design are common, many people are not sure if they should press it or just touch it. It gives feedback, so it is easy to know if you did something, but still the output is in Dutch, so it is hard to understand what exactly happened.

Satisfaction: The design is pleasant and easy once you have learned how to use it, but if you have no idea what to do, it gets annoying.
